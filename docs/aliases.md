# Aliases и собственные команды

Алиасы определены в `configs/zsh/aliases.zsh`; они работают только в
интерактивном Zsh. Стандартные `grep`, `find`, `du`, `ps`, `top`, `curl` и
`git` не заменяются, поэтому команды из документации и скриптов остаются
предсказуемыми.

| Команда | Назначение |
|---|---|
| `ls`, `la`, `ll`, `lt` | eza: список, скрытые, подробный вывод и дерево |
| `cat` | bat с подсветкой |
| `..`, `...`, `....` | перейти на 1, 2 или 3 уровня выше |
| `reload` | перезапустить Zsh (`exec zsh`) |
| `gs`, `ga`, `gc`, `gp`, `gd`, `gds`, `gl` | ежедневные команды Git |
| `lg` | открыть lazygit |
| `dc`, `dps`, `dcu`, `dcd`, `dcl` | Docker и Docker Compose |
| `k`, `kgp`, `kgs`, `kgd`, `kctx` | kubectl, если он установлен |
| `bp`, `bt` | btop и bottom |
| `th`, `the` | открыть или редактировать главную шпаргалку |

## Команды из `bin/`

`configs/zsh/exports.zsh` добавляет `bin/` репозитория в `PATH`.

```bash
mkcd ~/Development/new-service  # создать каталог и перейти в него (Zsh-функция)
ports                           # слушающие TCP-порты: lsof или ss
myip                            # один HTTPS-запрос, показать публичный IP
extract archive.tar.gz          # распаковать распространённый архив
git-clean-merged main           # только показать уже слитые локальные ветки
```

`git-clean-merged` намеренно ничего не удаляет. После проверки списка удаляйте
ветки вручную: `git branch -d <branch>`.

В неинтерактивном shell `mkcd` — отдельный исполняемый файл: он создаёт каталог
и печатает команду `cd`. Перейти в каталог родительскому процессу он не может;
в обычном Zsh работает удобная функция с тем же именем.
